Executive Summary

The codebase presents a clean FastAPI architecture with layered services, repositories, middleware, structured logging, Sentry hooks, Alembic migrations, and a Vite/React/Tailwind frontend. Dockerized dev/prod flows are present. The test suite is extensive. Overall readiness is high with specific gaps in secrets handling defaults, dependency management consistency, and a few production hardening areas.

Top 5 immediate actions:

  • Harden secrets and environment defaults (SECRET_KEY, ENCRYPTION_KEY, APP_DEBUG) and enforce at startup in production.
  • Unify Python dependency management (pyproject vs requirements) and enable reproducible lockfiles in CI/CD.
  • Pin and cache Playwright/browser installation in containers for faster, reproducible builds.
  • Add a minimal CI workflow to run lint/tests/build; ensure coverage threshold enforced.
  • Validate CORS origins and security headers for production, including CSP expansion when docs UI is disabled.

Security

Secrets and prod safety checks

High app/config.py defaults
  • secret_key and encryption_key have placeholder defaults.
  • app_debug default false, but no hard fail when prod values are insecure.

Recommendation: In Settings post-init, when is_production, enforce: app_debug == False, len(secret_key) >= 32, encryption_key not placeholder, and require cors_allowed_origins non-empty. Raise on violation to prevent boot.

Security headers coverage

Low app/middleware/security_headers.py
  • Strong defaults: X-Content-Type-Options, X-Frame-Options, Referrer-Policy, basic CSP, Permissions-Policy.
  • HSTS gated to production.

Recommendation: Expand CSP for API-only surface (e.g., default-src 'none'; frame-ancestors 'none') if no HTML is served; ensure WebSocket endpoints are compatible. Consider nonce/strict-dynamic if serving any HTML in future.

Dependencies & Tooling

Dual dependency manifests

Medium pyproject.toml vs requirements.txt
  • Both files exist; the Dockerfiles install via requirements.txt; tests/config use pyproject for tools.

Recommendation: Choose a single source of truth. Options:

  • Use pip-tools to compile locked requirements from pyproject, commit requirements.txt, and document update flow.
  • Or adopt Poetry/uv and rely solely on pyproject.lock in CI and Docker.
