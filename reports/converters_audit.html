Converters Audit Report

Scope: app/converters Generated:

Summary

Strengths

  • Clear abstraction via BaseConverter with explicit responsibilities.
  • Modular components: title optimization, description building, and marketplace-specific converter.
  • Safety: description HTML uses inline escaping for user-provided content to prevent XSS.
  • Title optimizer has a well-defined, incremental pipeline with analysis output for debugging.
  • eBay converter respects eBay constraints (80-char title, up to 12 images).

Key Risks / Gaps

  • Fixed condition="New" in eBay drafts; may be incorrect for some products.
  • SKU generation KI-<source_id> may collide across marketplaces or when IDs repeat.
  • DescriptionBuilder assumes product.source_marketplace is an enum with .value; may break if plain string.
  • URL/image safety relies on simple escaping; image URLs aren’t validated/sanitized.
  • Title abbreviations can be locale/vertical-sensitive; aggressive rules may reduce clarity in some categories.
  • No category mapping or item specifics population yet (mentioned as Phase 2).

Module Reviews

base_converter.py

  • Defines abstract interface for marketplace converters: convert, optimize_title, build_description.
  • Implements IConvertable contract from app.core.interfaces. Clean boundary.
  • Recommendation: Add type hints for return values already present; optionally document expected side-effect-free behavior.

description_builder.py

  • Provides three responsive, inline-CSS templates: Modern, Classic, Minimal; returns HTML strings.
  • Auto-extracts bullet features from description via regex (bullets, numbered lists, or sentences fallback).
  • Spec table built from brand/category/availability and displays source marketplace label.
  • Escapes user content (_escape) to mitigate XSS within text and attributes.
  • Assumes product.source_marketplace.value.title(); ensure compatibility if using strings.
  • Image URLs are only escaped; consider URL validation/whitelisting to prevent broken/malicious links.
  • Recommendation: Add alt text fallbacks, cap gallery size consistently (already caps to 4 extras), and optionally support more item specifics.

title_optimizer.py

  • Stepwise optimization: clean → remove marketplace noise → abbreviate → deduplicate → remove filler → truncate.
  • Provides optimize_with_analysis returning detailed TitleAnalysis metadata.
  • Abbreviations cover units and common terms (e.g., Wi‑Fi→WiFi, Stainless Steel→SS, Pack of N→N‑Pack).
  • Noise removal targets Amazon/Walmart promotional and badge text, trailing model codes, TM/®/© symbols.
  • Regex-based changes are global and context-agnostic; edge cases could over-abbreviate (e.g., brand/model names).
  • Recommendations:
    • Guard common brand terms from abbreviation via a protected-terms allowlist.
    • Expand stop-words for specific categories or languages behind configuration.
    • Expose max_length as a strategy parameter when marketplaces differ.

ebay_converter.py

  • Composes TitleOptimizer + DescriptionBuilder to generate ListingDraft for eBay.
  • Enforces eBay constraints: 80-char title, 12-image limit, sets TargetMarketplace.EBAY.
  • Uses fixed condition="New"; consider deriving from product attributes or defaulting to configurable value.
  • SKU generation KI-<source_product_id> is simple; consider namespacing by marketplace and normalizing.
  • Recommendation: Begin category mapping and item specifics population (e.g., brand, MPN, color, size) when available.

Recommendations (Prioritized)

  1. Introduce configurable condition mapping for eBay drafts; avoid always setting New.
  2. Harden SKU generation to ensure uniqueness across sources (e.g., EBAY-KI-<market>-<hash(source_id)>).
  3. Make source_marketplace handling robust in DescriptionBuilder: accept enums or strings safely.
  4. Validate/sanitize image URLs and optionally proxy them to ensure reliability and reduce mixed-content issues.
  5. Add protected-terms list in TitleOptimizer to prevent abbreviation of sensitive brand/model tokens.
  6. Add category mapping and basic item specifics extraction (brand, color, size, material) to improve listing quality.
  7. Provide response-model-like types or dataclasses for template outputs if later serialized beyond eBay HTML.
  8. Expand unit tests around edge cases: abbreviation collisions, unicode titles, long numeric model codes, minimal descriptions.

Appendix: Notable Patterns

  • Separation of concerns: optimization, description templating, and conversion are independently testable components.
  • Inline CSS ensures eBay compatibility; templates are mobile-friendly and lightweight.
  • Optimization pipeline is incremental and analysis-friendly, aiding troubleshooting and A/B testing.
