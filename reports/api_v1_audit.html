API v1 Audit Report

Scope: app/api/v1 Generated:

Summary

Strengths

  • Clear router structure per domain with coherent prefixes and tags.
  • JWT-protected endpoints; Stripe webhook signature verification.
  • Tenant isolation checks on resource fetch/update endpoints.
  • Good input validation with Pydantic models and constraints.
  • Rate limiting integrated on conversion endpoints with headers.
  • Resource hygiene around BrowserManager in conversion flow.

Key Risks

  • products.py scrape endpoint may not start/close BrowserManager properly.
  • OAuth state nonce in eBay connect is not persisted/validated server-side.
  • Frontend redirect URL inferred from CORS origins is brittle.
  • SSE path monkey-patches service methods; prefer explicit hooks.
  • Stripe webhook idempotency cache is in-memory only (per-process).
  • Some endpoints return dicts without response_model guarantees.

Module Reviews

auth.py

  • Endpoints: register, login, refresh, eBay connect/callback/status.
  • Maps domain errors to appropriate HTTP statuses; uses settings on refresh to embed current tier.
  • State parameter uses user_id:csrf_nonce but nonce is not persisted/validated. Persist with TTL to harden CSRF protection.
  • Use a dedicated frontend_base_url setting for redirects instead of deriving from CORS origins.
  • Tokens stored encrypted; expiry derived from expires_in. Consider persisting store_name if available.

billing.py

  • Endpoints: checkout, portal, subscription. Validates tier via regex.
  • Clear error mapping; logs unexpected failures.
  • Consider Stripe idempotency keys on checkout to avoid duplicate sessions on retries.

conversions.py

  • Endpoints: single convert, bulk JSON, preview, list, SSE bulk, job status/cancel, and debug eBay.
  • Rate-limited; sends WS events on completion; robust token refresh for eBay credentials.
  • Uses async context manager to start/close BrowserManager. Good.
  • SSE bulk converts via runtime monkey-patching of convert_url. Prefer explicit hook parameters (on_item_start/on_step/on_complete).
  • Ensure add_rate_limit_headers works with dict headers used for SSE, not only Response.

discovery.py

  • Endpoint: search with auth; clean exception mapping (400/429/500).
  • Searches are free (no rate limit). Documentation reflects that.

listings.py

  • Endpoints: list, get, update price, end listing. Enforces tenant isolation.
  • WS notifications on price updates and end actions. Good UX.

price_history.py

  • Endpoints: historical prices and aggregate stats, scoped to user products.
  • Shared helper _get_user_product provides validation and isolation.

products.py

  • Endpoints: scrape (preview conversion), list products, get product. Enforces tenant isolation.
  • Scrape endpoint constructs ConversionService(browser_manager=BrowserManager()) but does not start/close the browser. Align with conversions.py by wrapping in an async context that starts and closes the BrowserManager, or refactor to a common context helper.

users.py

  • Endpoints: profile, update profile, usage, admin set-tier. Uses UserService for updates.
  • Usage combines repo stats and tier limits from TIER_RATE_LIMITS.
  • Admin auth via X-Admin-Key equals secret_key; adequate but coarse-grained. Ensure upstream logging doesn’t capture this header.

webhooks.py

  • Stripe signature verification; in-memory LRU for idempotency; delegates to BillingService.
  • Emits WS tier-changed events; avoids logging payload secrets.
  • If horizontally scaled, use Redis/DB for event-id dedupe across nodes.

ws.py

  • WS auth via JWT query param; enforces access token type and per-tier connection limits.
  • Sends welcome and heartbeat; handles ping messages; cleans up on disconnect.
  • Validate ws_heartbeat_interval as numeric at config-load time (pydantic settings) to avoid runtime cast failures.

Recommendations (Prioritized)

  1. Fix BrowserManager lifecycle in products.py scrape endpoint to start/close browser around preview_conversion.
  2. Persist and validate OAuth state nonce (short TTL) for eBay connect/callback.
  3. Introduce frontend_base_url config and use it for redirects; avoid deriving from CORS origins.
  4. Refactor SSE bulk conversion to use explicit hooks rather than monkey-patching service methods.
  5. Back Stripe webhook idempotency with Redis/DB if running multiple replicas.
  6. Add response_model annotations for endpoints returning dicts to lock API contracts and enhance OpenAPI.
  7. Confirm add_rate_limit_headers works with header dicts and Response objects.
  8. Harden config validation (types/ranges) via pydantic settings; ensure ws_heartbeat_interval is numeric.
  9. Verify logging configuration never emits secrets (tokens, Stripe signatures) in any middleware.

Appendix: Notable Patterns

  • Tenant isolation via repository checks on user_id is consistently applied.
  • Best-effort WebSocket notifications avoid coupling API responses to push layer.
  • Token refresh path for eBay accounts persists new expiry and avoids mid-request expirations with a 5-minute buffer.

This report summarizes a static review of app/api/v1 as of generation time and may not reflect runtime configuration or external service availability.
