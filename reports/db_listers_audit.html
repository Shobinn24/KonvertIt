Database & Listers Audit Report

Scope: app/db and app/listers Generated:

Summary

Strengths

  • Clean async SQLAlchemy setup with pooled engine and slow-query instrumentation.
  • ORM models use SQLAlchemy 2.0 typing and timezone-aware DateTime.
  • Repositories implement tenant isolation patterns and focused query helpers.
  • Mappers bridge Pydantic domain models with ORM cleanly.
  • eBay lister follows official Sell Inventory API flow with robust error handling.
  • eBay OAuth manager encapsulates auth URL, code exchange, and refresh.

Key Risks / Gaps

  • Product.image_urls column type is JSON but annotated as dict; code writes lists. Align type hint.
  • Repository methods generally return all results; add hard caps or enforce upper bounds on limit.
  • Engine config assumes env-provided pool sizes; validate compatibility with target DB (e.g., Postgres vs. SQLite).
  • EbayLister policy/location auto-creation may have side effects in live accounts; consider feature flags or dry-run modes.
  • Inventory location creation uses default address; ensure this is configurable and not hardcoded.
  • EbayAuth scopes are broad; confirm principle of least privilege is met for listing-only operations.

Database Layer Review (app/db)

database.py

  • Creates async engine from settings with pool sizing, pre-ping, recycle, and echo in dev.
  • Attaches slow-query logger; session dependency commits/rolls back appropriately.
  • Recommendation: Add engine disposal hook on application shutdown to release connections.

models.py

  • Comprehensive schema: User, EbayCredential, Product, Conversion, Listing, PriceHistory, ProxyUsage.
  • Multi-tenant scoping via user_id and helpful composite unique index for product dedup.
  • Product.image_urls is declared as Mapped[dict] but default is list; should be Mapped[list] for consistency with usage.
  • Timezone-aware DateTime via timezone=True and UTC helpers; good consistency.
  • Recommendation: Consider ondelete behavior and cascades carefully (already present for many FKs).

mappers.py

  • Maps ScrapedProduct → Product and ListingDraft/Result → Listing; sets timestamps for completed conversions and active listings.
  • Handles None values and converts enum values correctly.
  • Recommendation: Ensure currency consistency when mapping price fields; consider storing currency on Product/Listing if used.

repositories/*

  • BaseRepository provides generic CRUD, count, get_all with optional user_id scoping.
  • ConversionRepository: user-scoped queries, status updates with converted_at set, linking listings, counts by status.
  • ListingRepository: find by user/status, by eBay ID, counts by status; helper for active listings.
  • ProductRepository: dedupe by (user, marketplace, source_id), filter by user/marketplace/brand.
  • PriceHistoryRepository: append-only record, range queries, stats, and latest retrieval.
  • Recommendation: Add result size limits and indexes alignment review based on query patterns (e.g., offers more composite indexes).

query_logger.py

  • Hooks into before/after cursor execute and logs slow queries with preview and duration.
  • Recommendation: Include parameter bindings in debug mode (sanitized) for deeper diagnostics.

Listers Review (app/listers)

ebay_auth.py

  • Generates auth URL with correct encoding (keeps :/ in scopes), exchanges codes, and refreshes tokens.
  • Uses Basic auth header for token endpoints; errors surface as EbayAuthError with response text in details.
  • Consider adding retries with backoff for transient HTTP failures and shorter timeouts with connect/read split.
  • Recommendation: Support sandbox vs production endpoints/scopes via configuration toggles.

ebay_lister.py

  • Implements full flow: inventory item → offer → publish. Provides update, end, and price update operations.
  • Maps condition strings to eBay enums and strips HTML for inventory product description.
  • Auto-manages business policies if not configured; ensures fulfillment shipFrom and inventory location presence.
  • Auto-creating policies/locations can be surprising in production; introduce feature flags and explicit setup guidance.
  • Default inventory address is hardcoded; move to settings and validate.
  • Recommendation: Add idempotency or cleanup around partial failures; more granular error parsing from eBay errors array.

Recommendations (Prioritized)

  1. Align Product.image_urls typing with usage (use list in ORM annotation and migrations if needed).
  2. Parameterize inventory location defaults (address, key) and guard policy/location auto-creation behind a feature flag.
  3. Add bounds and safe defaults to repository list queries (limit ≤ 500) to prevent unbounded fetches.
  4. Add engine disposal on shutdown; ensure pool sizes match DB limits in production.
  5. Add retry/backoff to eBay OAuth/token and API requests where transient failures are common.
  6. Expand DB indexes based on real query patterns (e.g., user_id+created_at for listings/conversions pagination).
  7. Enhance slow query logs with sanitized parameters in development for easier optimization.

Appendix: Notable Patterns

  • Tenant isolation via repositories and user_id scoping is consistently applied.
  • Inventory API flow adheres to eBay guidelines, including offer cleansing of stale entries.
  • Schema uses timezone-aware timestamps for auditable events.
