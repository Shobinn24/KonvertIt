Core Module Audit Report

Scope: app/core Generated:

Summary

Strengths

  • Clear domain modeling with Pydantic and well-scoped enums for marketplaces, statuses, tiers, and risk levels.
  • Consistent custom exception hierarchy enabling clean error handling and Sentry tagging.
  • Structured logging via structlog with environment-aware JSON/console rendering.
  • Practical resilience patterns: circuit breaker and async retry with backoff+jitter.
  • Health checks concurrently probe DB and Redis with latency metrics and non-disruptive 200 responses.
  • Encryption uses Fernet with lazy initialization and backward-compatibility for legacy plaintext.

Key Risks / Gaps

  • Fernet key validity assumed; lack of startup validation can defer failures to runtime.
  • Health check always returns 200 (by design); ensure external monitors consider body status.
  • Retry decorator defaults to retrying on Exception; risk of masking programmer errors.
  • Models: ScrapedProduct.scraped_at uses naive datetime; consider timezone-aware.
  • Circuit breaker window/threshold may not be thread-safe if accessed across tasks (single instance expected).
  • Sentry dropping 4xx is good; verify rate limits/429 from upstream are not lost if needed for observability.

Module Reviews

encryption.py

  • Fernet-based symmetric encryption with lazy cipher init from settings.
  • Graceful fallback on InvalidToken to return raw value (supports legacy plaintext tokens).
  • Add startup validation to ensure settings.encryption_key is a valid Fernet key to fail-fast.
  • Consider key rotation strategy (versioned keys) if secrets policy requires it.

exceptions.py

  • Hierarchical exceptions grouped by domain (Scraping, Conversion, Listing, Resilience).
  • ComplianceViolationError enriches message and stores metadata.
  • Works well with Sentry filter to tag KonvertItError subclasses.

health.py

  • Asynchronous DB and Redis probes with latency metrics and warning logs on failure.
  • Overall status is healthy when all probes up, otherwise degraded.
  • Returns 200 regardless; ensure load balancer logic relies on body content if needed.
  • Recommendation: Add optional probe for external dependencies (e.g., Stripe reachability) via head checks with timeouts.

interfaces.py

  • Defines core ABCs: IScrapeable, IConvertable, IListable, IPriceable, IComplianceCheckable.
  • Methods documented with expected errors and contracts; promotes loose coupling and testability.
  • Recommendation: Consider Protocols (typing.Protocol) where structural typing is useful for simpler mocking.

logging_config.py

  • Configures structlog with shared processors and chooses JSON vs console based on env/flag.
  • Attaches ProcessorFormatter to root handler so stdlib logging is structured.
  • Quiets noisy loggers (uvicorn.access, sqlalchemy.engine, httpx) to WARNING.
  • Recommendation: Mask secrets in logs (e.g., tokens) via a custom processor if any may leak from upstream middlewares.

models.py

  • Defines source/target marketplaces, statuses, tiers, and domain models for pipeline and business logic.
  • Validation constraints: title lengths, price bounds, images max length (12), etc.
  • Datetime fields (e.g., scraped_at, created_at) are naive; consider timezone-aware (UTC) or consistent serialization policy.
  • Suggestion: Provide model_config (pydantic v2) to control json_encoders/timezone, and forbid_extra to harden inputs if used for request parsing.

resilience.py

  • Implements CircuitBreaker with CLOSED/OPEN/HALF_OPEN and cooldown logic; async context manager pattern is ergonomic.
  • Retry decorator with exponential backoff and jitter; logs failures and delays.
  • Default retryable_exceptions=(Exception,) risks retrying on programmer errors; recommend narrowing to I/O classes.
  • Recommendation: Expose hooks/metrics for breaker state changes and retry attempts (Prometheus counters).

sentry_config.py

  • Initializes Sentry with FastAPI/Starlette/asyncio/logging integrations; configurable tracing rates.
  • before_send filter drops 4xx HTTPExceptions and tags KonvertItError types; avoids PII.
  • Recommendation: Add environment-based sampling (higher in staging, lower in prod) and denylist paths (e.g., health) if noisy.

Recommendations (Prioritized)

  1. Validate encryption key at startup (fail-fast) and consider key rotation support.
  2. Tighten retryable exceptions to network/HTTP/timeout classes to avoid retrying on logic errors.
  3. Adopt timezone-aware datetimes (UTC) in models, or enforce consistent serialization via config.
  4. Expose breaker and retry metrics; optionally add per-source configs for thresholds and cooldowns.
  5. Allow health endpoint to optionally return 503 in readiness mode (separate from liveness) if dependencies are down.
  6. Add a logging processor to scrub secrets/tokens from messages and contexts.
  7. Consider Protocols for interfaces to ease testing and reduce ABC boilerplate for simple adapters.

Appendix: Notable Patterns

  • Clean separation of concerns across error handling, logging, resilience, and domain modeling.
  • Non-intrusive health model: never 503, encourages downstream to inspect body status for degradation.
  • Resilience components are composable and side-effect aware via async contexts and decorators.
